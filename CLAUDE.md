# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

BIST (Borsa Istanbul) Discord bot with two independent entry points: (1) a news bot (`main.py`) that polls RSS feeds + KAP disclosures, scores news relevance by ticker/keyword matching, deduplicates, and pushes high-scoring items to a Discord **webhook**; (2) an interactive gateway bot (`bot.py`) serving the `/hisse` slash command (price + candlestick chart + indicators + technical outlook + analyst consensus) and a `/health` command (live news-source diagnostics). All comments, log messages, and the README are in Turkish — keep that convention when editing.

## Commands

```bash
pip install -r requirements.txt
python main.py             # haber botu — requires DISCORD_WEBHOOK_URL in .env
python bot.py              # /hisse komut botu — requires DISCORD_BOT_TOKEN in .env
python charting.py THYAO 1a  # grafik debug: _chart_THYAO.png yazar, Discord gerekmez
```

No tests, linter, or build system. Runtime config lives in `.env` (see README table): `POLL_INTERVAL_SECONDS`, `MIN_RELEVANCE_SCORE`, `ENABLE_KAP`, `ENABLE_PRICE`, `RSS_FEEDS`, `RUN_ONCE` (single cycle then exit — used by GitHub Actions), `SEEN_DB_PATH`.

An empty `seen.db` triggers a **silent first round**: items are marked seen but not sent (prevents a spam burst on fresh deploys or cache misses). So deleting `seen.db` does NOT cause re-sends — it causes one quiet cycle instead. Rows older than `RETENTION_DAYS` (60, in store.py) are pruned on startup, so the db never grows unbounded; the dedup data is deliberately disposable (a remote DB was considered and rejected — single instance, self-healing loss).

Deployment: `.github/workflows/bot.yml` runs the bot on a ~10-min cron via `RUN_ONCE=1`, carrying `seen.db` between runs with `actions/cache` (immutable keys + `restore-keys` prefix trick). `DISCORD_WEBHOOK_URL` comes from repo Actions secrets; `ENABLE_KAP=0` there because KAP's WAF blocks datacenter IPs (it blocks home IPs too as of June 2026 — see `fetch_kap`'s warn-once handling). Deploying = pushing to GitHub; the next cron tick picks up the new code. Keep workflow action versions on Node 24-capable majors (checkout@v5, setup-python@v6, cache@v5).

## Architecture

Single asyncio polling loop in `main.py` runs the pipeline every `POLL_INTERVAL_SECONDS`:

```
sources.fetch_rss / fetch_kap  →  filters.evaluate (score)  →  store.SeenStore (dedup)
                               →  inference.infer (rule-based impact)  →  notifier.enrich_prices (optional)
                               →  notifier.build_embed + send (Discord)
```

- **`filters.py` owns the shared `NewsItem` dataclass** — `sources.py` imports it from there, so `filters` must stay import-free of `sources` to avoid a cycle. Sources fill the raw fields; `evaluate()` mutates the item in place to fill `tickers`, `score`, `category`, `matched_keywords`.
- **Scoring** = ticker bonus (`_TICKER_BONUS` per matched ticker, capped at `_TICKER_BONUS_CAP`) + summed `KEYWORD_WEIGHTS`. Items below `MIN_RELEVANCE_SCORE` are still marked seen (so noise isn't re-evaluated) but not sent. The dominant keyword category picks the embed color (`CATEGORY_COLORS`).
- **Turkish-aware matching is mandatory**: `textnorm.fold()` lowercases and ASCII-folds Turkish characters (Python's `.lower()` mishandles `İ`). All keyword patterns and ticker aliases are folded at build time and matched against folded text. Any new matching logic must go through `fold()`; never compare raw text. Patterns in `SENTIMENT_PATTERNS` (inference.py) are therefore written in ASCII form ("temettu", not "temettü").
- **`inference.py`** produces a rule-based impact estimate (`Inference`: direction pozitif/negatif/karisik/belirsiz + matched reason labels + expected targets). Targets are the matched tickers, or "BIST geneli" for ticker-less makro news. Returns `None` when there's nothing to show. Rendered in the embed as "📌 Olası Etki (tahmini)".
- **Dedup** keys on `NewsItem.uid` (RSS entry id/link, or `kap-<index>`), stored in SQLite `seen.db` via `store.SeenStore`.
- **`sources._clean()` strips HTML tags AND iteratively unescapes entities** — some feeds double-encode (`&amp;#039;` → `&#039;` → `'`), so it loops until stable. Applied to both title and summary. Don't bypass it when adding fields from feed entries.
- **KAP source is currently dead**: KAP's new Next.js site put `kap.org.tr/tr/api/disclosures` behind a Citrix WAF that blocks bots (timeouts / custom "666" error pages — verified June 2026; not bypassable with headers/cookies). `fetch_kap` warns once then drops to DEBUG (`_kap_uyarildi` module flag), and logs recovery if access returns. Dead sources (RSS or KAP) are always skipped, never crash the bot. The block is path-specific blackholing of `/tr/api/disclosures` (the connection just hangs → timeout), NOT a TLS/header issue — `curl_cffi` Chrome-impersonation is also blackholed there even from a residential IP, while the normal KAP pages load fine (re-verified June 2026). The data is fetched client-side, so only a headless browser could scrape it — and that won't help the Actions deployment anyway (datacenter-IP block). Decision: leave KAP off and compensate via RSS — `DEFAULT_RSS_FEEDS` includes a borsa-focused feed (borsagundem) that republishes borsa/KAP news. (Its sister site finansgundem.com serves near-identical content under different URLs, so it was NOT added — dedup keys on URL and wouldn't catch the cross-site dupes.)
- **`notifier.py`** handles Discord 429 rate limits with retry, and fetches prices via `yfinance` (blocking, so wrapped in `asyncio.to_thread`) with a 60s in-memory cache. `_parse_published()` normalizes RSS/KAP date strings (RFC 2822, ISO 8601, epoch ms, `dd.mm.yyyy HH:MM`) into the embed's `timestamp` field so Discord renders local time; unparseable dates fall back to raw text in the footer. Naive datetimes are assumed Turkey time (UTC+3).
- **Embed is deliberately minimal** (user asked for readability): no score/category fields (color still encodes category), footer is source-only, date/time via the native `timestamp` field, tickers listed inside the "Olası Etki" field — a separate price field appears only when prices were actually fetched. Don't re-add metadata fields.

### `/hisse` command bot (second entry point, independent of the news pipeline)

`bot.py` is a discord.py 2.x **gateway** client (always-on process, `DISCORD_BOT_TOKEN`), entirely separate from the webhook news loop — it must NOT be wired into `main.py` or the Actions cron (a 10-min wake/exit cycle can't answer interactions; hosting decision deferred, runs locally for now). It imports `textnorm`/`tickers` read-only, plus `sources`/`store` read-only for the `/health` diagnostic — a deliberate diagnostic-only coupling that does NOT run the news loop.

```
/hisse kod:<autocomplete> periyot:<1 Hafta|1 Ay|3 Ay|1 Yıl>  →  market.fetch_history (2y daily)
   →  market.compute_indicators (SMA50/SMA200/MACD/RSI, full series)  →  market.slice_window (5/22/66/252 bars)
   →  market.get_overview (price, volume, floatShares, analyst consensus)  →  analiz.teknik_gorunum (rule-based outlook, full series)
   →  charting.render_chart (PNG BytesIO)
   →  embed (overview + 📈 Teknik Görünüm + 🎯 Analist Konsensüsü) + attachment://grafik.png + GrafikView in ONE followup.send
```

- **Period buttons** — the followup carries a `GrafikView` (discord.ui.View) with one button per `market.PERIYOT_SIRA` entry. The full computed `df` + `ov` + `gor` (teknik görünüm) are stashed in the view; a click only re-slices + re-renders the chart (NO refetch/recompute — the outlook and analyst fields are **period-independent**, computed on the full series, so they're reused as-is) and `edit_original_response`s the message in place (active period button is `primary`+disabled). Anyone in the channel can switch; the view times out after 10 min and disables its buttons. `market.PERIYOT_MUM`/`PERIYOT_SIRA` are the single source of truth for periods — both the slash `choices` and the buttons derive from them, so adding a period is a one-line dict edit.

- **`market.py`** — all functions are blocking (yfinance/pandas); `bot.py` wraps every call in `asyncio.to_thread` (same pattern as `notifier.enrich_prices`). Indicators are plain pandas (deliberate: no TA-Lib/pandas-ta dep) computed on the FULL ~2y series, sliced to the display window AFTER — never compute on the slice. `fast_info` keys are camelCase (`lastPrice`); `get_info()` (floatShares/marketCap, slow + rate-limited) sits behind a 6h in-memory cache (`_info_cache`) and degrades to `None` fields on failure — the command never dies on missing info. Invalid tickers return an EMPTY DataFrame (no exception): gate on `market.is_valid`. `compute_indicators` also computes **SMA50** (for `analiz.teknik_gorunum`'s crossover signal). `get_overview` reads the **analyst consensus** (`recommendationKey`, `numberOfAnalystOpinions`, `targetMean/High/LowPrice` + computed `hedef_potansiyel`) from the SAME cached `info` dict — zero extra request; raw values are returned (Türkçe label + `<2 analist` guard live in `bot.py`).
- **`analiz.py`** — rule-based technical outlook (the price-side counterpart to `inference.py`). `teknik_gorunum(df)` runs on the FULL ~2y indicator series (NOT the slice — same warmup warning as the indicators) and returns a `TeknikGorunum` (yon: Güçlü Yükseliş/Yükseliş/Nötr/Düşüş/Güçlü Düşüş · skor · guven düşük/orta/yüksek · nedenler · uyari). It sums weighted `(puan, etiket)` signals from `_sinyaller` (Close vs SMA200, SMA50↔SMA200 crossover, Close vs SMA50, MACD histogram sign, RSI zones/slope, ~20-bar return); `guven` is signal alignment, hard-capped to "düşük" with a caveat when <200 bars (SMA200 not warmed up). Returns `None` on too-little data. `python analiz.py KOD` debug path. Pure pandas (+`fmt`), discord-free.
- **`charting.py`** — `matplotlib.use("Agg")` MUST precede any pyplot/mplfinance import. **3 panels** (candles+SMA50/SMA200 / MACD / RSI) — the volume panel was removed (user request) to give candles more vertical room. Every `make_addplot` passes `secondary_y=False` or mplfinance silently moves series to a right-hand axis (bit us: RSI 30/70 guides landed on their own scale). Renders via `returnfig=True` then `fig.savefig` so the panels get post-draw cosmetics in `_suzle_panelleri` (MACD zero line, shaded RSI 70-100/0-30 zones + 30/50/70 guides, SMA50/SMA200 value labels); that cosmetic pass is wrapped in try/except so it can never break the command. **Two MAs on the price panel**: **SMA50** (blue, short MA) is folded into `_hesapla_ylim`'s candle range so it's always in its true position (no clamp — it sits near price); **SMA200** (yellow) is "always visible" but special-cased — ylim includes it when within reach, else keeps a candle-focused ylim (candles must stay above `_MIN_MUM_ORANI` 0.35) and clamps the line to the near edge with a `▲`/`▼` + real-value label. `_sma_etiketleri` draws both right-edge value labels and nudges them apart if they'd vertically overlap. MACD histogram bars are coloured per-sign (green/red). `python charting.py KOD 1h|1a|3a|1y` is the chart-aesthetics debug loop (`_chart_*.png`, gitignored).
- **`bot.py`** — `Intents.none()` + `guilds=True` only (no privileged intents). `GUILD_ID` env set → instant per-guild command sync (dev); empty → global sync (~1h propagation). Every code path after `defer()` must end in `followup.send` or the interaction hangs. Autocomplete searches TICKERS codes AND aliases via `fold()`; codes outside TICKERS are accepted (BIST ~600 vs 99 listed) and validated by data presence; company-name input falls back to `find_tickers()`. `fmt.py` formats all numbers Turkish-style (`1.234.567,89`, `Mr`/`Mn` TL) — never emit raw `f"{x:,}"`. `_overview_embed` adds two full-width fields below the overview via `_teknik_alani` (📈 Teknik Görünüm — emoji per `gor.yon`, top-3 nedenler, optional ⚠️ uyari; omitted when `gor is None`) and `_analist_alani` (🎯 Analist Konsensüsü — `_TAVSIYE_TR` maps `recommendationKey` to Güçlü AL/AL/TUT/Endeks Altı/SAT, shows mean target + `hedef_potansiyel` + range; **hidden entirely when `<2` analysts**, and shows "Tavsiye: —" + target-only when key is "none" but a target exists).
- **`/health` command** — live diagnostic for the news pipeline (the news bot is a separate process, so this **probes sources from the command bot's own environment**, not the Actions run). `sources.probe(session, feeds, enable_kap)` fetches each RSS feed + KAP **concurrently** (`asyncio.gather`) via `_probe_rss`/`_probe_kap` and returns per-source dicts (`tip/ad/url/ok/adet/sure_ms/hata`) — these never raise and are independent of `fetch_kap`'s warn-once state. Feed list comes from `sources.feeds_from_env()` (shared with `main._cfg`, single source of truth). `bot.py` renders ✅/❌ per source + a `store.db_stats()` summary (read-only `mode=ro` connect — never creates/prunes the db, unlike `SeenStore()`). Embed colour: green all-ok / orange partial / red all-down. (`/health` already earned its keep: it caught that the old `bigpara.hurriyet.com.tr/rss/` default feed had died with HTTP 404 — replaced with `hurriyet.com.tr/rss/ekonomi`.)

## Extension points (as designed)

- New stocks: `TICKERS` dict in `tickers.py` (code → folded aliases), or bulk-load via `load_from_csv()`. Aliases are matched by substring on folded text — pick aliases carefully to avoid false positives (see the `"bim "` trailing-space trick).
- Scoring tweaks: `KEYWORD_WEIGHTS` in `filters.py` — `(keyword, weight, category)` tuples; keywords are written in normal Turkish and folded automatically.
- Impact inference: `SENTIMENT_PATTERNS` in `inference.py` — `(folded_regex, direction, weight, label)` tuples; weight 0 entries only contribute a reason label without steering direction.
- Technical-outlook tuning: `analiz._sinyaller` — each signal appends a `(puan, etiket)` tuple; adjust the weights, RSI bands (70/30), or the ±5% return threshold there. The yon thresholds (±2 / ±5) live in `teknik_gorunum`.
- LLM classification: `filters.llm_classify()` is a deliberate stub for a future Anthropic API call; rule-based scoring/inference is the default path (user explicitly chose rule-based over LLM for inference).
- New source types: add an async fetcher in `sources.py` returning `list[NewsItem]` and call it from `run_once()` in `main.py`.

## Decisions already made (don't re-propose)

- Rule-based inference over LLM (cost); LLM remains a stub.
- `/hisse` technical outlook (`analiz.py`) is **rule-based**, consistent with the news-inference decision (rule + hybrid-LLM + full-LLM were offered; rule-based chosen). It outputs a **qualitative label + reasons**, deliberately NOT a fake "% rise probability" (misleading without a validated model — explicitly declined).
- Analyst data is yfinance's **anonymized consensus** (rating + mean target + analyst count), pulled from the already-cached `get_info` (zero extra request). Named Turkish-broker calls ("İş Yatırım: AL, hedef X") were declined: they'd require scraping (fragile + ToS risk + the KAP-WAF scar). The field is **hidden below 2 analysts** to drop stale/unreliable single-analyst targets (e.g. a target that's a multiple of the current price).
- Keyword-only passes stay: news with no BIST ticker match can still be sent on keyword score alone (e.g. foreign-company "halka arz" news). A require-ticker filter was offered and declined.
- No remote DB for dedup: single instance + disposable data + silent first round make SQLite-in-cache sufficient.
- `/hisse` is ONE slash command with a `kod` parameter, not per-ticker commands (Discord caps global slash commands at 100; BIST has ~600 tickers).
- Command bot is NOT hosted on GitHub Actions (cron can't serve interactions; 24/7 Actions jobs are a ToS gray area). **Hosting: a persistent VM (chosen: Oracle Cloud Always Free) running `bot.py` under `systemd` with `Restart=always`** — see `deploy/` (the `bist-hisse-bot.service` unit, `update.sh`, and `README.md` setup guide). `main.py` stays on the Actions cron; the two bots deploy independently. `git push` only redeploys the news bot (next cron tick); the gateway bot is updated on the VM via `deploy/update.sh` (git pull + pip + `systemctl restart`).
- Single `requirements.txt` for both entry points (Actions needlessly installs discord.py/matplotlib for the news bot, but pip cache makes it cheap; a split file was considered and rejected for simplicity).
